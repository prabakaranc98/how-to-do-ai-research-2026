# Background Index

**Sprint:** 001 - Epistemic Reliability in Small Language Models

This folder collects the background needed to think clearly before implementing experiments.

The goal is not endless literature review. The goal is to build enough intuition, notation, theory, and method awareness to design a focused research sprint.

## Files

- `01-foundations.md`: conceptual foundation, first-principles framing, uncertainty lenses, and how to think about the problem.
- `02-notation-and-metrics.md`: symbols, variables, evaluation setup, calibration metrics, selective prediction, conformal framing, and decision metrics.
- `03-methods-and-theory.md`: practical methods and theory tools for uncertainty, retrieval agreement, abstention, routing, and failure analysis.
- `04-related-work-map.md`: related-work areas to study, what each area contributes, and what may still be missing.

## How To Use This Folder

Use this folder as the research scratchpad before implementation:

1. Read the foundations to get the mental model.
2. Use the notation file to make the experiment measurable.
3. Use the methods file to choose the first small set of uncertainty signals.
4. Use the related-work map to avoid reinventing obvious ideas and to identify gaps.
5. Keep adding notes as the sprint evolves.

The rule:

**Study only enough to sharpen the next experiment.**
